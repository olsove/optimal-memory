# Aggregate and sort on less data

## Slow <Turtle strokeWidth={1.75} />

```sql
SELECT customer_id, SUM(amount)
FROM payments
GROUP BY customer_id
ORDER BY SUM(amount) DESC;
```

---

## Fast <Rabbit strokeWidth={1.75}/>

Reduce rows before aggregating:

```sql
SELECT customer_id, SUM(amount)
FROM payments
WHERE created_at >= '2026-03-01'
GROUP BY customer_id
ORDER BY SUM(amount) DESC;
```

---

## Why <Search strokeWidth={1.75}/>

Sorting and aggregating large datasets increases:

- Memory usage
- CPU usage
- Temp files

Filtering early reduces the workload.

---

## Typical Improvement <Settings strokeWidth={1.75}/>

`3x-50x`

---

## Best Use Cases <BookOpen strokeWidth={1.75}/>

- Financial reporting
- Analytics dashboards
- Business intelligence queries
- Time-range reports

## Test it out yourself <TestTubeDiagonal strokeWidth={1.75}/>

### 1. Set up Docker Database
Run a local Docker instance with PostgreSQL 18:

```bash
docker run --name pg18-test -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:18-alpine
```

### 2. Create Schema and Sample Data
Run this script to create the `payments` table and generate sample data:
> User: postgres
> Password: postgres

```sql
CREATE TABLE payments (
    id SERIAL PRIMARY KEY,
    customer_id INT NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    created_at TIMESTAMP NOT NULL
);

-- Generate 1,000,000 payments
INSERT INTO payments (customer_id, amount, created_at)
SELECT
    (random() * 1000 + 1)::int,
    (random() * 500)::numeric(10,2),
    NOW() - (random() * interval '365 days')
FROM generate_series(1, 1000000);

CREATE INDEX idx_payments_created_at ON payments(created_at);
```

#### Fast (Filter before aggregate) <Rabbit strokeWidth={1.75}/>
Filtering rows first significantly reduces the amount of data that needs to be grouped and summed.
```sql
EXPLAIN (ANALYZE, BUFFERS)
SELECT customer_id, SUM(amount)
FROM payments
WHERE created_at >= '2026-03-01'
GROUP BY customer_id
ORDER BY SUM(amount) DESC;
```
