# Write index-friendly predicates

## Slow <Turtle strokeWidth={1.75} />
For date/time filtering:
```sql
SELECT *
FROM events
WHERE DATE(created_at) = '2026-03-16';
```

---

## Fast <Rabbit strokeWidth={1.75}/>
For date/time filtering:
```sql
SELECT *
FROM events
WHERE created_at >= '2026-03-16'
  AND created_at < '2026-03-17';
```

For case-insensitive filtering, use an expression index if needed:

```sql
CREATE INDEX idx_users_lower_email
ON users (LOWER(email));
```

Then:

```sql
SELECT *
FROM users
WHERE LOWER(email) = 'alice@example.com';
```

---

## Why <Search strokeWidth={1.75}/>

Functions on indexed columns often disable index usage.
Rewrite predicates so PostgreSQL can use index range scans.
(Or use expression indexes.)

---

## Typical Improvement <Settings strokeWidth={1.75}/>

`10x-100x`

---

## Best Use Cases <BookOpen strokeWidth={1.75}/>

- Timestamps
- Email lookups
- Search filters
- Reporting queries

## Test it out yourself <TestTubeDiagonal strokeWidth={1.75}/>

### 1. Set up Docker Database
Run a local Docker instance with PostgreSQL 18:

```bash
docker run --name pg18-test -e POSTGRES_PASSWORD=postgres -p 5432:5432 -d postgres:18-alpine
```

### 2. Create Schema and Sample Data
Run this script to create the `events` table and generate sample data:
> User: postgres
> Password: postgres

```sql
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    created_at TIMESTAMP NOT NULL,
    data JSONB
);

-- Add an index on created_at
CREATE INDEX idx_events_created_at ON events(created_at);

-- Generate 1 million events over the last year
INSERT INTO events (created_at, data)
SELECT
    NOW() - (random() * interval '365 days'),
    '{"info": "test"}'::jsonb
FROM generate_series(1, 1000000);
```

### 3. Test the Performance

#### Slow (Function on Column) <Turtle strokeWidth={1.75} />
This forces a Sequential Scan because the index on `created_at` cannot be used with the `DATE()` function.
```sql
EXPLAIN ANALYZE
SELECT *
FROM events
WHERE DATE(created_at) = CURRENT_DATE - INTERVAL '1 day';
```

#### Fast (Index-Friendly Range) <Rabbit strokeWidth={1.75}/>
This uses an Index Scan because the column itself is used in the predicate.
```sql
EXPLAIN ANALYZE
SELECT *
FROM events
WHERE created_at >= CURRENT_DATE - INTERVAL '1 day'
  AND created_at < CURRENT_DATE;
```
