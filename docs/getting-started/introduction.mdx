# My Ultimate Performance Guide

Welcome to my ultimate performance guide.

After years of experience, I've developed a comprehensive guide that will help you write and maintain performant queries.

This guide is designed to be a comprehensive resource for developers who are looking to improve their database performance.

It covers various aspects of database optimisation, including indexing strategies, query optimisation techniques, and best practices for writing efficient SQL queries.


## When it comes to performance, where do developers go wrong?

> Most slow queries happen because they:
>
> - Scan too many rows
> - Sort too many rows
> - Repeat the same work many times
> - Prevent index usage

> Fixing the query shape usually gives the biggest improvement.
